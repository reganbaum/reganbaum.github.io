Accessibility Testing

For this lab, I used several web accessibility evaluation tools to see how accessible my midterm website was. In testing my website, I found two main problems with my site.

First, the header nesting patterns in my website do not follow the web accessibility guidelines. Only <h1> and <h2> header tags should follow an <h1> tag, but I only use <h1> and <h3> tags throughout my site; I skipped over the <h2> tag. To fix this, I would change all the <h3> tags to <h2> tags so that the header levels follow the accessibility guidelines.

Second, my midterm website does not have any page regions or ARIA landmarks. Web pages should have regions defined, especially for the main content, but I mostly only used <div> tags. To fix this, I would make the HTML tags on my pages more specific. For example, I would add a <mav> tag to my navigation links. I would also add <main> tags to the main content area of each page.

As I build the website for my final project, I will keep these ideas in mind and focus more on building an accessible website. I will also regularly check it to ensure that the website follows the accessibility guidelines.
